# NetWatch Light — security & robustness rework (2026-06-11)

Branch: `feat/fable-rework-christoph`

This rework hardens the existing FastAPI + vanilla-JS "light" build. It does **not**
change the deployment model (still a single FastAPI process with JSON-file state);
the larger production architecture in `TECHNICAL_AUDIT.md` (arq worker, TimescaleDB,
Redis, Docker) is intentionally left as a separate, larger effort (see *Deferred*).

## Findings addressed

### Critical
- **Stored XSS via SNMP/LLDP data.** The legacy renderers wrote device/interface/
  alert/event strings into `innerHTML` without escaping. `sysName`, `ifAlias`,
  `ifDescr` and LLDP neighbour fields are controlled by the monitored (potentially
  compromised or spoofed) devices, so this was a real stored-XSS sink. All such
  renderers now HTML-escape their inputs (`renderDashboardDevices`, `renderEvents`,
  `renderDevicesTable`, `renderDeviceDetail`, `renderTopology`, `renderAlerts`,
  `renderSettings`, and `renderKpis` — which re-injected the event-stream status).
- **No auth on mutating endpoints.** All mutating `POST` routes now accept an optional
  setup token (`X-Setup-Token` / `NETWATCH_SETUP_TOKEN`), per the project's own
  security decision in `TECHNICAL_AUDIT.md`. Read-only routes and the WS stay public.
- **Plaintext credential file.** The state file (which stores SNMP communities and
  SNMPv3 keys in this build) is now written `0600` with its parent dir `0700`.
  At-rest encryption remains a deferred production item.

### High / correctness
- **WebSocket died on first disconnect.** Added reconnect with exponential backoff
  (1s→15s) plus an error handler. Previously a single backend restart left the UI
  permanently stale until a manual reload.
- **Unthrottled snapshot refetch race.** Each WS message triggered a full
  `/api/snapshot` fetch with no de-duplication; parallel responses could land out of
  order and clobber newer state. Refetches are now coalesced (single in-flight +
  one queued).
- **Deprecated FastAPI events.** `@app.on_event("startup"/"shutdown")` replaced with a
  `lifespan` context manager.
- **Missing error handling.** `runPoll`, `runDiscovery`, `clearLiveInventory` and the
  polling toggles now have `try/catch` with user-visible feedback.

### Cleanup
- Removed `prototype/` (a duplicate, fully-unescaped earlier UI — a second XSS surface;
  the backend only ever served `web/`).
- Removed dead client-side auto-poll code (`startAutoPoll`/`stopAutoPoll`, never called).
- Dot-tunnel animation now pauses when the tab is hidden (was a permanent rAF loop).
- Backend split into `config`, `auth`, `models`, `fixtures` modules; `main.py`/`state.py`
  slimmed.

## Tests
`tests/` (pytest): state lifecycle + persistence permissions, SNMP parsing helpers,
API smoke, setup-token enforcement, env-driven config. Run:
`pip install -r requirements-dev.txt && pytest tests/ -q`.

## Deferred (would need sign-off — new stack/dependencies)
- Move polling to a separate `arq`/Redis worker (the in-process scheduler is explicitly
  flagged "vietato per produzione" in `TECHNICAL_AUDIT.md`).
- TimescaleDB-backed metric history instead of JSON state.
- At-rest encryption + key management for SNMP credentials.
- Frontend modularization (`web/app.js` is one large file) and a JS test runner.
- Known pre-existing, runtime-safe type imprecision in
  `state._propagate_endpoint_port_traffic` (`link.get("from_interface")` may be `None`);
  left untouched to keep this PR surgical.
